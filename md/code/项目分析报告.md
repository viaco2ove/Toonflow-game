# Toonflow 项目分析报告

> 分析时间：2026-04-04

---

## 一、业务定位

这是一个 **AI 短剧/互动故事游戏一站式工具平台**，核心业务分两条主线：

### 主线 1：AI 短剧制作流水线
> 小说文本 → 大纲 → 剧本 → 分镜 → AI 图像/视频 → 成片

| 业务模块 | 说明 |
|---|---|
| 小说管理 (`novel`) | 导入、管理小说章节文本 |
| 大纲生成 (`outline`) | AI 从小说提取剧情大纲、故事线 |
| 剧本生成 (`script`) | 按大纲生成结构化对白剧本 |
| 分镜制作 (`storyboard`) | 生成分镜提示词、画面设计 |
| 视频合成 (`video`) | 调用 AI 视频 API（豆包/Sora 等）合成片段 |
| 素材管理 (`assets`) | 图像/视频素材库 |

### 主线 2：多角色 AI 故事游戏（近期重点）
> 世界观创建 → 章节设计 → 游戏会话 → 角色互动 → 叙事引擎驱动

这是项目现在最复杂的核心，`src/routes/game/` 有 29 个接口，`src/modules/game-runtime/` 是专属运行时引擎。
特点：ai 故事的现状就是无法达到理想化的无限输入和无限上下文和无限token 和极速返回。所以要进行信息取舍和拆分。

## 四、代码特点

### 1. 路由自动生成系统（core.ts）
- 用 `fast-glob` 扫描 `src/routes/**/*.ts`，自动按文件路径映射为 HTTP 路由
- 用 **MD5 哈希**判断是否需要重新写 `router.ts`，避免不必要的磁盘写入
- 类似 Next.js 文件系统路由，但是运行在 Express 上

### 2. 游戏状态机设计（gameEngine.ts ~1900行）
- `ChapterProgressState`：章节进度（阶段 ID、事件 Index、用户节点状态）
- `RuntimeDynamicEventState`：动态事件（AI 实时生成的叙事摘要和事实）
- `evaluateCondition()`：支持自然语言条件 + JSON 条件表达式（and/or/not/equals/contains/gt...）
- 所有状态读写通过纯函数（`readXxx`/`setXxx`/`upsertXxx`），状态以 JSON 存入 SQLite

### 3. 叙事引擎驱动（NarrativeOrchestrator）
- 基于 AI SDK（Vercel AI SDK + 多模型适配）流式生成对话
- 按 Phase（阶段）→ Event（事件）→ Turn（回合）的三层结构推进剧情
- 支持：玩家输入 → 触发器检查 → 叙事推进 → 记忆总结 → 快照持久化

### 4. 多 AI 模型适配

```
OpenAI / DeepSeek / Anthropic / Google / xAI / 阿里通义 / 智谱 / 本地兼容
```

使用 `@ai-sdk/*` 统一接口，并记录 Token 用量和费用（`t_aiTokenUsageLog`）

### 5. 双运行模式

| 模式 | 说明 |
|---|---|
| 纯 API Server | `yarn dev`，Express on 60002 端口 |
| Electron 桌面端 | `yarn dev:gui`，内嵌 Chromium，前后端一体打包 |

### 6. 数据库：SQLite + Knex
- 所有数据用 SQLite 单文件存储（`db.sqlite`）
- Knex 作 Query Builder，无 ORM，schema 在 `initDB.ts` 代码定义
- 支持按需增量建表（`fixDB.ts`）

## 七、总体架构特点总结

| 特点 | 说明 |
|---|---|
| **单体全栈** | 后端 API + 前端静态文件 + Electron 壳，一个仓库可打包桌面应用 |
| **文件系统路由** | routes/ 目录即路由，自动发现，无需手动注册 |
| **纯函数状态机** | 游戏状态用纯函数读写，利于测试和回放 |
| **状态全存 JSON in SQLite** | 灵活但无严格 Schema 约束，适合快速迭代 |
| **多 AI 模型 + 流式输出** | WebSocket + Server-Sent Events 实时推送叙事 |
| **双业务支柱** | 短剧制作流水线（旧） + AI 互动游戏引擎（新主线） |
| **轻中间件** | 仅 JWT 鉴权 + 资源隔离，业务逻辑直写 routes |

## 八、关键文件导读

1. **`gameEngine.ts`（~1900 行）** —— 游戏状态机的核心数据层，定义所有状态结构和纯函数操作，是读懂游戏模块的关键入口
2. **`core.ts`** —— 路由自动生成机制，新建 `src/routes/xxx/yyy.ts` 文件路由即自动注册，类似 Next.js 但运行在 Express 上
3. **`NarrativeOrchestrator.ts`** —— 叙事引擎核心，控制 AI 对话流程、回合推进和记忆管理
4. **`SessionService.ts`** —— 游戏会话核心服务，连接路由层与引擎层的枢纽
5. **`initDB.ts`** —— 所有数据库表结构定义，了解数据模型的唯一入口

---

> **总结**：这是一个从 **AI 短剧工具** 向 **沉浸式 AI 故事游戏平台** 战略转型的项目。后端以 Express + SQLite 为基础快速迭代，游戏运行时部分（NarrativeOrchestrator + ChapterProgressEngine）是目前最核心、最复杂的技术投入点。README 里短剧功能已标为"old"，游戏功能（`routes/game/` 29个接口）是现在的发展主线。
