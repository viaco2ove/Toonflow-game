## 章节内容转换为多个事件的完整流程

根据代码分析，我现在可以完整回答你的问题：**章节内容是如何转换为多个事件发给编排师的**

### 1. 章节内容的结构

章节内容使用**Markdown格式**，通过二级标题(`##`)来划分不同的阶段：**示例**：
```markdown
## 全局状态（仅用户可见）
@系统：你已被卷入【混沌裂隙】...

## 场景开始
@旁白：破碎的星辰残骸漂浮在虚空之中...

## 🧩 用户行动1（被编排触发）
@系统（仅对用户）：你被冲击波掀飞...

## 主线推进
@旁白：战斗愈发激烈...

## 🧩 用户行动2（异常感知）
@系统（仅对用户）：⚠️ 检测到异常...

## 非事件：时间停滞
@旁白：爆炸达到顶点...
```



### 2. 转换流程（代码实现）

**核心函数**：`extractRuntimeSections()` (gameEngine.ts 第310-327行)

```typescript
function extractRuntimeSections(input: unknown): Array<{ heading: string; body: string }> {
  const text = normalizeEditorText(input);
  const headingRegex = /^##\s*(.+)$/gm;  // 匹配二级标题
  const sections: Array<{ heading: string; body: string }> = [];
  const matches = Array.from(text.matchAll(headingRegex));
  
  for (let i = 0; i < matches.length; i += 1) {
    const current = matches[i];
    const next = matches[i + 1];
    const bodyStart = (current.index ?? 0) + current[0].length;
    const bodyEnd = next?.index ?? text.length;
    sections.push({
      heading: String(current[1] || "").trim(),  // 标题文本
      body: text.slice(bodyStart, bodyEnd).trim(), // 标题下的内容
    });
  }
  return sections;
}
```

**转换逻辑**：
1. **识别阶段**：通过正则表达式`/^##\s*(.+)$/gm`匹配所有二级标题
2. **提取内容**：每个标题之间的内容作为一个独立的section
3. **生成Phase对象**：每个section转换为一个`ChapterRuntimePhase`对象

### 总结

**章节内容转换为多个事件的完整流程**：

1. **内容解析**：章节Markdown内容 → 通过`##`标题分割 → 多个section
2. **Phase生成**：每个section → 转换为`ChapterRuntimePhase`对象（包含id/label/kind/targetSummary等）
3. **状态存储**：所有Phase存储在章节的`runtimeOutline`字段（JSON格式）
4. **运行时读取**：根据`progress.phaseId`从`state`中找到当前Phase
5. **事件构建**：当前Phase → 构建`currentEvent`上下文（包含eventIndex/eventKind/eventSummary等）
6. **AI调用**：将`currentEvent`信息打包成payload → 传递给编排师AI

**关键点**：
- **一个Phase = 一个事件**（在编排流程中）
- **Phase类型**：`opening`（开场）、`scene`（场景）、`user`（用户行动）、`fixed`（固定事件）
- **事件流转**：通过`ChapterProgressEngine`管理Phase之间的跳转（支持条件判断、前置依赖）
- **AI提示词**：编排师收到的不是完整章节内容，而是当前Phase的摘要和上下文信息
