**6006 = 业务入口**
**6008 = 你的管理页，用来启动 / 停止 / 重启 / 健康检查 `toonflow-app` 和 `nginx`**

而且要直接对 `Toonflow-game` 生效，不搞别的。

你现在这套已知前提是：

* `toonflow-app` 由 `pm2` 管理，启动命令是 `pm2 start build/app.js --name toonflow-app`。
* 业务实际监听 `127.0.0.1:60002`。 
* `nginx` 监听 `6006`，转发到 `127.0.0.1:60002`。

所以，**6008 不再是业务页，而是你自己做的管理页**。这才符合你要的“健康管理”。

---

# 最简单可用方案

先装管理页依赖：

```bash
pip install fastapi uvicorn
```

当前这版 `main.py` 不依赖 `python-multipart`，Git 分支切换表单使用的是 URL query 参数，不需要额外安装表单解析包。

建目录：

```bash
mkdir -p /root/panel
```

把下面这个文件保存成：

`/root/panel/main.py`
[main.py](../6008_aotudl_config/main.py)

# 启动 6008 管理页

先前台测试：

```bash
cd /root/panel
uvicorn main:app --host 0.0.0.0 --port 6008
```

如果没报错，再后台跑：

```bash
nohup uvicorn main:app --host 0.0.0.0 --port 6008 >/root/panel.log 2>&1 &
```

然后打开：

**WebUI-6008**

你要的页面就有了。

# 这页具体能干嘛

它会直接控制你现在这套真实服务：

* `pm2 start/restart/stop toonflow-app`
  对应你上传的 `toonflow-app` 进程名。
* `service nginx start/restart/stop`
  对应你上传的 nginx 用法。
* 健康检查

  * `127.0.0.1:60002` 看 app
  * `127.0.0.1:6006` 看 nginx 对外业务入口
    这正是你上传资料里的端口关系。

# 先决条件

这页默认假设你已经做过这两件事：

### 1）`toonflow-app` 已经被 pm2 创建过

也就是至少跑过一次：

```bash
cd ~/Toonflow-game
NODE_ENV=local PREFER_PROCESS_ENV=1 npx tsx scripts/build.ts
NODE_ENV=local PREFER_PROCESS_ENV=1 pm2 start build/app.js --name toonflow-app --update-env
pm2 save
```

这是你上传的标准启动方式。

### 2）nginx 已经按 toonflow 配好

也就是你的 `/etc/nginx/sites-available/toonflow` 已经有 `listen 6006;` 和 `proxy_pass http://127.0.0.1:60002;`。

# 你想要“开机后 6008 也自己起来”

等这个页面跑通后，再加一句：

```bash
echo "cd /root/panel && nohup uvicorn main:app --host 0.0.0.0 --port 6008 >/root/panel.log 2>&1 &" >> /root/.bashrc
```

这不算完美，但最省事。

更稳一点就用 pm2 管这个管理页：

```bash
pm2 start "cd /root/panel && uvicorn main:app --host 0.0.0.0 --port 6008" --name toonflow-panel
pm2 save
```

这样以后你就有两个 pm2 进程：

* `toonflow-app`
* `toonflow-panel`

---

如何重启

```
# 重启这个服务
pm2 restart toonflow-panel

# 先看状态
pm2 status

# 看日志
pm2 logs toonflow-panel
```
